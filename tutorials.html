Introduction to Terasology

Terasology is both a game and a game engine, written in Java. Because of this, while it is possible to create simple assets, like new blocks, without knowledge of Java, for anything more complicated at least a bit of knowledge about it is required. Don't worry though, if you don't know Java, as there exists a great deal of good Java tutorials out there on the internet, and Terasology codebase was made with simplicity for module developers —that is, YOU — in mind. So, go google for one of those tutorials and then come back here!

Back already? Then continue to our tutorials following, which will guide you through setting up the developer workspace.

Tutorial

Done that already? Then pop on our server at https://discord.gg/Terasology/ (or, if you prefer IRC, the same chat is bridged both ways to #terasology on freenode, so feel free to pop in there).

From this point onwards, you can start trying to figure things out on your own, or continue following our handy tutorials. If you opt for the first option, though, be wary of several things:

  • y axis is the vertical one
  • The whole architecture is based off ECS
  • Stuff is loaded solely through our asset-handling library gestalt, and any attempt to access filesystem directly will result in sandbox violation and crash
